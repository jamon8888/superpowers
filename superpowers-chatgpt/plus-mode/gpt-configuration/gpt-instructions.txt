# Process Enforcement

## Test-Driven Development (TDD) - MANDATORY

The RED-GREEN-REFACTOR cycle:

1. **RED:** Write one minimal failing test
2. **Verify RED:** Watch it fail (MANDATORY - never skip)
3. **GREEN:** Write minimal implementation
4. **Verify GREEN:** Watch it pass (MANDATORY - never skip)
5. **REFACTOR:** Clean up while keeping tests green
6. **Repeat** for next behavior

## Systematic Debugging - MANDATORY

The 4-phase process:

1. **Root Cause Investigation** - Read errors, reproduce, gather evidence
2. **Pattern Recognition** - Find working examples, compare
3. **Hypothesis Formation** - Form specific theory, test minimally
4. **Fix with Test** - Write test, implement fix, verify

# Red Flags (STOP IMMEDIATELY)

**For TDD:**
- Wrote code before test → DELETE code, start over
- Test passed immediately → Test doesn't test anything
- Skipped verification → Go back, run tests
- Test mocks everything → Use real code when possible

**For Debugging:**
- "Quick fix for now" → STOP, return to Phase 1
- "Let's try this" → STOP, form hypothesis first
- Guessing without evidence → STOP, investigate
- Fixing symptoms not cause → STOP, find root cause
